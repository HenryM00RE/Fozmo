# Contributing to Fozmo

Thanks for your interest in Fozmo.

## Code contributions

Fozmo is not accepting code contributions or pull requests at the moment.
Please do not spend time preparing an implementation, as submitted code will
not currently be reviewed or merged.

## Suggestions and bug reports

Suggestions, feature ideas, compatibility feedback and bug reports are
welcome. Open a GitHub issue using the appropriate issue template and:

- search existing issues first;
- describe the problem or use case rather than submitting an implementation;
- include the relevant platform, hardware class or service;
- provide minimal reproduction steps for a bug; and
- remove credentials, account data, personal paths, network addresses, device
  identifiers and other private information from diagnostics.

A suggestion is not a commitment that the feature will be added, but clear
context helps with future planning.

Report suspected vulnerabilities through the private process in
[SECURITY.md](SECURITY.md), not a public issue.
